• First version 1986
  • Created by Joe Armstrong
  • Acronym for Ericsson Language
  • Built for the fault-tolerant, distributed needs of telecom
  • Behind CouchDB and Riak
  • Prolog syntax
  • Lisp paradigm
  • No objects
  • Immutability
  • Usually deterministic
  • Complied
  • Dynamically typed
Concurrency
  • Runs on a VM (BEAM)
  • Lightweight processes instead of threading
  • Message passing via actors
Reliability
  • “Let it crash.” — Erlang Mantra
  • Limited error checking
  • Easy to watch for killed processes
  • Hot swap code
